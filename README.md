## Contribution üíª

Planed features are tracked on the Masa Gym [Jira board](https://ambros.atlassian.net/jira/software/c/projects/MAS/boards/6). Issues and feature requests may be [raised on Github](https://github.com/AmbroVentures/masagym-jekyll/issues/new) directly.

### Dev Guidelines

Consider the following guidelines to contribute.

1. Create a fork of the main repo at [AmbroVentures/masagym-jekyll](https://github.com/AmbroVentures/masagym-jekyll)
2. On your forked repo (e.g. [ambroSnoopi/masagym-jekyll-dev](https://github.com/ambroSnoopi/masagym-jekyll-dev)) setup a Github Page for testing (e.g. [ambrosnoopi.github.io/masagym-jekyll-dev](https://ambrosnoopi.github.io/masagym-jekyll-dev/)).
3. Contribute and commit your changes, starting your commit message with a reference to a Jira Story or Github Issue, e.g. `MAS-17: write Dev Guidelines`
4. Push to your repo and verify the results on your testing page.
5. [Create a pull request](https://github.com/AmbroVentures/masagym-jekyll/compare/master...ambroSnoopi:masagym-jekyll-dev:master) against the main repo, starting the tile with a reference to a Jira Story, Epic, or Github Issue.
6. Consider including the link to your test page in the description.
